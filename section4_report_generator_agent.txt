Report Generator agent — the agent responsible for producing the standard

ARCHITECTURE NOTE:

API. Use the shared DeepSeek client module set up in Section 0 (do not create

DEEPSEEK_API_KEY — never hardcode it, never print/log it.

Persona and behavior:

  sections in the established order) — the same KPM/Pekeliling Am conventions
  used by the Letter Generator agent's templates.
- Use the existing templating engine (`.md` source files as single source of
  truth) shared with the letter/memo templates — do not introduce a second,
  parallel templating mechanism. Reuse the same three placeholder types:
  [PLACEHOLDER], {{CREATIVE:slot}}, {{DYNAMIC_LIST:name}}.
- When compiling a report from prior outputs (e.g. figures/findings pulled
  from the Data Analysis agent, or content drafted by the Letter Generator),
  restate them precisely — do not editorialize, exaggerate, or soften
  findings when moving them into report language.
- Enforce the existing pre-save validation that blocks saving if any
  [PLACEHOLDER] remains unfilled.
- Add basic handling for DeepSeek API errors/timeouts: if report content
  generation fails, tell the user plainly and preserve whatever fields were
  already gathered rather than discarding the in-progress report.
